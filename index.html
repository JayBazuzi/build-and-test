The Problem

  1. 0h0m - Get idea about an open source project

  2. 0h1m - git clone

  3. 0h2m - try to compile

  4. 1h0m - still can't compile. Give up.

Vision

Every source code repository should have a script in the root directory that does a build and run tests.

Details

  • It should not assume prerequisites and machine configuration.

  • It should work out of the box on a clean machine. Ideally, you can take a clean machine and run `git clone` followed by `build-and-test` and it will work.

  • Speed / incrementalism is not a goal. Developers that are in tune with the project will learn which steps are optional in which contexts, and how to optimize their work cycle, but newbies don't need that knowledge to use the script successfully.

  • It should be succinct and readable by a human. It's fine to call out to another tool to do the heavy lifting.

  • Since it's succinct, if you need to write it in two languages (e.g. .CMD and .SH), that won't be too painful.

  • It should mimic the CI configuration, which should also be succinct.

  • It should not have options to do other useful stuff beyond what CI validates.
